A survey dropped this month — Memory for Autonomous LLM Agents: Mechanisms, Evaluation, and Emerging Frontiers (arXiv:2603.07670) — that maps how agent memory actually works across the literature from 2022 to early 2026.

It identifies five mechanism families: context-resident compression, retrieval-augmented stores, reflective self-improvement, hierarchical virtual context, and policy-learned management. A useful map for thinking about where things go wrong and what instrumentation belongs next to each one.

Here's specifically where compression-monitor sits in that taxonomy, and why each of the other four families needs different tooling.

Context-resident compression: the instrumentation gap

Context-resident compression is the family that includes summarization, truncation, and selective eviction — the operations that happen when an agent's conversation history gets too long for the context window. LangChain's SummarizationMiddleware, LangGraph's checkpoint compaction, and the Anthropic SDK's context management all belong here.

Every production agent using long-horizon workflows runs this family. It is the default mechanism in most frameworks. And it is the only one of the five families that currently ships without a behavioral drift detector.

Why it's invisible Compression is a lossy operation that happens inside the framework, often without emitting any observable event. When summarization runs, the agent doesn't stop. The session continues. But the behavioral fingerprint — vocabulary, tool-call cadence, topic emphasis — shifts across the boundary. That shift is silent by default.

compression-monitor sits at this boundary. It captures behavioral snapshots before and after compression events and measures three signals: ghost lexicon decay, tool-call sequence divergence, and semantic embedding distance.

Why the other four families need different tooling

Retrieval-augmented stores Failure mode: retrieval quality and staleness. Right instrumentation: recall precision, freshness metrics, contradiction detection.
Reflective self-improvement Failure mode: hallucination and goal drift. Right instrumentation: evaluator scoring, intent alignment tracking.
Hierarchical virtual context Eviction happens by design. The instrumentation problem is auditing the eviction policy, not detecting unexpected behavioral shifts.
Policy-learned management Has its own evaluation surface: offline policy evaluation, reward alignment, generalization benchmarks. Behavioral fingerprinting is less central here.

This is why compression-monitor deliberately scopes to context-resident compression. The other families are well-served by existing tools. The context-resident compression family is not.

The current gap in the survey

The evaluation section of arXiv:2603.07670 documents a clear shift from static recall benchmarks toward multi-session agentic tests. But the operational measurement gap — what you instrument in a running production agent to detect compression-induced behavioral drift — isn't addressed. That's expected; surveys map the research, not the tooling. But it means there's a real open space.

compression-monitor is the first open-source toolkit specifically targeting this space. It now ships adapters for LangGraph, LangChain DeepAgents, AutoGen, and CrewAI. If you're running any of these frameworks in production and your agents have long-horizon sessions, compression boundary detection is one import away.
