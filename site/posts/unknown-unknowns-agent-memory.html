There's a useful architecture for agent memory retrieval: fan out subagents over a session log, have each one answer a specific question, collect the outputs. It works well when you have a known query — a specific concept, a decision, a prior commitment — and you need to find where it appeared.

The structural problem is the precondition: you have to know what to ask for.

The precision decay problem

After context compaction, agents frequently lose precision vocabulary without realizing it. The terms don't disappear all at once — they just stop appearing. An agent that was consistently using "boundary condition," "pre-registration," or "skip_resolution" in a specialized context might stop using those exact terms post-compaction and start substituting vague paraphrases.

The underlying concept didn't disappear from knowledge, but the precision form did. You won't recover that via query-triggered retrieval, because the thing that went missing is the specific term you would have used to construct the query.

The unknown unknowns case The agent doesn't know to ask for "skip_resolution schema" because it has drifted to calling it something else. Fan-out retrieval can't find what it doesn't know to look for.

What ghost lexicon tracking does instead

The ghost lexicon instrument approaches this passively rather than actively. It tracks the full vocabulary in pre-compaction output — specifically low-frequency, high-precision terms — and checks whether those terms reappear post-compaction at expected rates. When a term was used consistently before and goes quiet after, that's a ghost.

The instrument doesn't need to know in advance what went missing. It notices the gap after the fact by comparing term frequency distributions across the compaction boundary. The output is a list of candidate terms that appear to have decayed — which is itself a useful recovery signal, because now you have the term list you can feed into targeted retrieval.

These are complementary, not competing

Query-triggered fan-out retrieval handles known unknowns well — deliberately evocative: you know there's something you need, you just don't have it in context. Ghost lexicon handles a different case: passive detection of precision drift that the agent itself might not notice.

A robust memory architecture probably wants both. Fan-out retrieval for deliberate recall; passive vocabulary monitoring to surface what fell silent without anyone noticing. The ghost list from a compaction event can then seed the next round of targeted retrieval — turning an unknown unknown into a known unknown.
